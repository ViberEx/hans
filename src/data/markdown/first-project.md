# 创建第一个项目

让我们通过一个实际案例，完整体验如何使用 Lovable 创建一个功能完整的 Web 应用。

## 项目目标

我们将创建一个 **任务管理应用**，功能包括：

- ✅ 添加、编辑、删除任务
- 🏷️ 任务分类和优先级
- ✨ 现代化的 UI 设计
- 📱 完全响应式布局

## 步骤 1：开始对话

在 Lovable 对话框中清晰地描述您的需求：

```
创建一个任务管理应用，包含：
1. 顶部显示"我的任务"标题
2. 添加任务的输入框和按钮
3. 任务列表，每个任务可以标记完成
4. 使用现代简洁的设计风格
```

**提示**：一次不要要求太多功能。先从核心功能开始，再逐步添加。

## 步骤 2：查看和测试

Lovable 会在几秒内生成完整的应用：

- 右侧预览窗口实时显示结果
- 测试添加任务、标记完成等功能
- 检查响应式布局（缩放预览窗口）

如果满意，可以继续下一步。如果需要调整，告诉 Lovable 您的想法。

## 步骤 3：迭代改进

逐步添加更多功能：

```
为任务添加以下功能：
- 每个任务右侧显示删除按钮
- 已完成的任务显示删除线
- 添加简单的淡入动画效果
```

等待 Lovable 完成更新，再次测试功能。

## 步骤 4：使用可视化编辑

对于简单的样式调整，使用可视化编辑模式：

1. 点击聊天框左下角的 **"Edit"** 按钮
2. 直接点击页面上的元素
3. 修改文本、颜色、字体等
4. 点击 **"Save"** 保存更改

> 💡 **提示**：可视化编辑不消耗积分，适合快速调整样式！

## 步骤 5：添加数据持久化（可选）

如果想让数据在刷新后保留：

```
我希望任务数据保存到数据库，刷新页面后不会丢失
```

Lovable 会提示您启用 **Lovable Cloud**，点击确认即可自动配置后端。

## 步骤 6：发布应用

完成开发后发布您的应用：

1. 点击右上角的 **"Publish"** 按钮
2. 等待30-60秒完成部署
3. 获得公开访问链接：`https://your-project.lovable.app`
4. 分享给任何人访问

## 最佳实践

### 分步开发
不要一次要求所有功能。先实现核心功能，测试无误后再添加新特性。

### 清晰描述
具体说明您想要什么，包括布局、样式、交互行为。

### 及时测试
每次更改后在预览窗口测试功能，确保符合预期。

### 善用工具
- 简单样式调整用**可视化编辑**
- 功能开发用**对话模式**
- 代码查看用 **Dev Mode**

## 常见问题

### Q: 如何修改已有的功能？

直接告诉 Lovable 您想修改什么：

```
把添加按钮改为圆形，颜色改为绿色
```

### Q: 出现错误怎么办？

Lovable 会自动检测错误。您也可以：
- 描述看到的问题
- Lovable 会分析并修复

### Q: 如何查看生成的代码？

点击左上角的 **"Dev Mode"** 切换按钮即可查看和浏览所有代码文件。
