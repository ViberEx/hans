# 可视化编辑 (Visual Edits)

Visual Edits 是 Lovable 的强大功能，让您可以直接在页面上选择元素并快速修改，无需编写提示词，大部分操作还完全免费！

## 什么是 Visual Edits

Visual Edits 允许您：
- 直接点击选择页面上的元素
- 即时修改文本、颜色、字体（**完全免费**）
- 使用提示词对选中元素进行更复杂的修改
- 实时预览所有更改
- 一次性保存所有修改

## 启用 Visual Edits

### 方法 1：通过聊天框
1. 在聊天框左下角找到 **"Edit"** 按钮
2. 点击激活 Visual Edits 模式
3. 页面上的可选元素会高亮显示

### 方法 2：快捷键（即将推出）
使用键盘快捷键快速切换编辑模式

### 退出编辑模式
- 点击 **"Cancel"** 取消所有未保存的修改
- 点击 **"Save"** 保存修改并退出
- 按 ESC 键快速取消

## 可以编辑什么

### ✅ 可选择的元素（静态元素）

- **文本内容**: 标题、段落、按钮文字、标签
- **图片**: 静态图片（非动态生成的）
- **背景**: 背景颜色、渐变
- **容器**: div、卡片、区块
- **导航**: 导航栏项目、链接
- **按钮**: 所有静态按钮
- **输入框标签**: 表单标签文字

### ❌ 不可选择的元素（动态元素）

- **列表项**: 通过循环生成的重复内容
- **动态内容**: 从数据库或 API 加载的内容
- **条件渲染**: 根据逻辑显示/隐藏的元素
- **动态组件**: 根据状态动态创建的组件

> 💡 **如何判断**: 如果一个元素无法选中，它很可能是动态生成的，需要通过提示词来修改。

## 编辑方式

### 1. 直接编辑（免费 🎉）

选中元素后，可以在右侧面板直接修改：

#### 文本编辑
- **内容**: 直接修改文字内容
- **字体大小**: 调整字号（px、rem、em）
- **字重**: 设置粗细（100-900）
- **颜色**: 使用颜色选择器或输入 HEX/RGB 值

#### 样式编辑
- **背景颜色**: 修改背景色
- **边框**: 颜色、宽度、样式
- **圆角**: 设置 border-radius
- **阴影**: 添加 box-shadow

#### 布局编辑
- **内边距 (Padding)**: 设置元素内部间距
- **外边距 (Margin)**: 设置元素外部间距
- **对齐方式**: 左对齐、居中、右对齐
- **宽度和高度**: 调整尺寸

> 🎉 **完全免费**: 直接编辑不消耗任何积分！这是 Visual Edits 最强大的特性。

### 2. 提示词编辑（消耗积分）

选中元素后，也可以在输入框中输入提示词：

```
把这个按钮改成圆角，添加悬停时的阴影效果，背景色改成渐变
```

**特点**:
- 可以实现更复杂的修改
- 可以一次修改多个属性
- 每条提示词消耗 1 个积分
- 适合需要添加动画、交互等复杂需求

### 3. 组合使用

最高效的方式是组合使用：
1. 用提示词创建基础结构和复杂功能
2. 用 Visual Edits 免费调整文本、颜色、间距
3. 用提示词添加动画和交互效果

## 实际使用场景

### 场景 1：快速调整文案

**需求**: 修改首页标题文字

**操作步骤**:
1. 点击 Edit 按钮
2. 选中标题元素
3. 在文本框中修改文字
4. 点击 Save

**优势**: 免费、快速、无需消耗积分

### 场景 2：调整配色方案

**需求**: 更换应用的主题色

**操作步骤**:
1. 启用 Visual Edits
2. 依次选中主要元素（按钮、导航栏等）
3. 使用颜色选择器调整颜色
4. 一次性保存所有更改

**优势**: 可视化选色，实时预览效果

### 场景 3：微调布局

**需求**: 调整卡片之间的间距

**操作步骤**:
1. 选中容器元素
2. 调整 padding 或 margin 数值
3. 实时查看间距变化
4. 满意后保存

**优势**: 精确控制间距，所见即所得

### 场景 4：添加复杂动画

**需求**: 为按钮添加悬停动画效果

**操作步骤**:
1. 选中按钮元素
2. 在输入框输入：
   ```
   添加悬停时向上移动 2px 的动画，同时增加阴影
   ```
3. AI 生成动画代码
4. 保存修改

**优势**: 复杂效果也能轻松实现

## 最佳实践

### 1. 优先使用直接编辑

对于以下修改，**务必使用直接编辑**而不是提示词：
- ✅ 修改文字内容
- ✅ 调整字体大小和颜色
- ✅ 更换背景颜色
- ✅ 调整内外边距
- ✅ 修改边框和圆角

**原因**: 完全免费且更快捷！

### 2. 批量修改相似元素用提示词

如果要修改多个相似的元素：

```
把所有的按钮都改成圆角 8px，添加悬停时上浮的效果
```

这比一个个选中并手动修改更高效。

### 3. 保存前充分预览

在 Visual Edits 模式下：
- 所有修改都是实时预览的
- 可以随时撤销（Cancel）
- 确认无误后再 Save

### 4. 理解静态 vs 动态

**静态元素示例**:
```tsx
<h1>欢迎来到我的网站</h1>  ✅ 可以用 Visual Edits
<Button>点击这里</Button>   ✅ 可以用 Visual Edits
```

**动态元素示例**:
```tsx
{products.map(product => (
  <Card key={product.id}>     ❌ 需要用提示词
    {product.name}
  </Card>
))}
```

### 5. 合理安排工作流

推荐的开发流程：

1. **第一阶段 - 结构搭建**: 用提示词创建页面结构
2. **第二阶段 - 细节优化**: 用 Visual Edits 调整样式
3. **第三阶段 - 功能完善**: 用提示词添加交互
4. **第四阶段 - 最后润色**: 用 Visual Edits 微调细节

## 常见问题

### Q: 为什么有些元素选不中？

**A**: 该元素是动态生成的，例如：
- 通过 `.map()` 循环渲染的列表项
- 通过条件判断显示的内容
- 从数据库加载的动态内容

**解决方法**: 使用提示词修改：
```
把商品列表中的标题改成加粗，字号改大
```

### Q: 修改可以撤销吗？

**A**: 
- 未保存前：点击 Cancel 可以取消所有修改
- 已保存后：可以通过版本历史回滚（在编辑器中查看历史）

### Q: Visual Edits 会生成代码吗？

**A**: 会的！保存后，Lovable 会自动更新对应的代码文件。点击 Dev Mode 可以查看生成的代码。

### Q: 移动端能用 Visual Edits 吗？

**A**: 目前 Visual Edits 主要针对桌面端优化，移动端体验可能受限。建议在桌面端使用。

### Q: 可以编辑组件库的组件吗？

**A**: 可以编辑 shadcn/ui 组件的实例，但如果要修改组件的默认样式，需要用提示词修改组件文件。

### Q: 直接编辑真的免费吗？

**A**: 是的！Visual Edits 的直接编辑功能（修改文本、颜色、字体、间距等）完全免费，不消耗任何积分。只有使用提示词编辑时才会消耗积分。

## 高级技巧

### 技巧 1: 快速配色

使用颜色选择器的高级功能：
1. 点击颜色选择器
2. 使用取色器从页面其他地方取色
3. 或输入精确的 HEX 值
4. 调整透明度创建半透明效果

### 技巧 2: 对齐和间距检查

Visual Edits 可以帮助检查设计一致性：
1. 启用编辑模式
2. 选中元素可以看到元素的边界
3. 检查不同元素的间距是否一致
4. 确保对齐是否准确

### 技巧 3: 响应式调整

虽然 Visual Edits 主要在桌面端使用，但可以：
1. 在桌面端修改元素
2. 切换到移动端预览查看效果
3. 如果需要移动端特殊样式，使用提示词：
   ```
   在移动端，把这个标题字号改小，间距改为 16px
   ```

### 技巧 4: 设计微调阶段

在应用基本完成后，使用 Visual Edits 进行最后的打磨：
- 统一字号和字重
- 确保间距一致
- 优化颜色对比度
- 微调元素对齐

这个阶段基本不需要消耗积分！

### 技巧 5: 复制样式值

在 Visual Edits 中查看元素的样式值：
1. 选中元素
2. 查看右侧面板的样式属性
3. 记录或复制这些值
4. 用于其他元素的一致性设计

## Visual Edits 与提示词对比

| 功能 | Visual Edits | 提示词 |

| 修改文本 | ✅ 免费、快速 | ❌ 消耗积分 |
| 调整颜色 | ✅ 免费、可视化 | ❌ 消耗积分 |
| 修改间距 | ✅ 免费、精确 | ❌ 消耗积分 |
| 添加动画 | ❌ 不支持 | ✅ 支持 |
| 修改布局 | ⚠️ 简单布局可以 | ✅ 复杂布局更好 |
| 批量修改 | ❌ 需要逐个选择 | ✅ 一次修改多个 |
| 动态元素 | ❌ 不支持 | ✅ 支持 |
| 添加功能 | ❌ 不支持 | ✅ 支持 |
