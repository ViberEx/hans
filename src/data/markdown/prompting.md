# 提示词技巧

掌握提示词技巧是高效使用 Lovable 的关键。好的提示词能让 AI 更准确地理解您的需求，更快地构建出理想的应用。

## 核心原则

### 1. 具体明确

AI 无法读懂您的想法，需要清晰的描述。

❌ **不好的例子**：
```
做一个网站
```

✅ **好的例子**：
```
创建一个个人博客网站，包含：
- 顶部导航栏（首页、文章、关于我）
- 文章列表页，显示标题、摘要、日期
- 文章详情页，支持 Markdown 格式
- 使用简洁现代的设计风格
```

### 2. 分步进行

一次构建一个功能，而不是要求整个应用。

❌ **不好的例子**：
```
创建完整的电商平台，包含首页、商品列表、详情页、
购物车、结账、支付、用户中心、订单管理、库存管理、
评论系统等所有功能
```

✅ **好的例子**：
```
第一步：创建商品列表页，显示商品卡片（图片、名称、价格）

[测试后继续]

第二步：添加商品详情页，点击商品卡片可以查看详情

[测试后继续]

第三步：实现购物车功能
```

### 3. 描述视觉和交互

不仅说要什么，还要说明它应该如何呈现和运作。

✅ **包含视觉和交互细节**：
```
创建一个登录表单：
- 居中显示在页面中
- 白色卡片，圆角 12px，轻微阴影
- 包含邮箱和密码输入框
- 蓝色主按钮"登录"
- 点击后显示加载状态
- 错误时在表单下方显示红色提示信息
```

## 高级技巧

### 引用设计参考

提及知名网站或应用的设计风格：

```
创建一个类似 Airbnb 的搜索栏：
- 简洁的圆角设计
- 悬浮时显示阴影
- 包含位置、日期、人数三个输入区域
- 右侧是搜索按钮（圆形，红色）
```

### 使用设计术语

如果熟悉设计概念，可以直接使用专业术语：

```
创建仪表板页面：
- 使用 CSS Grid，三列布局，gap 为 24px
- 卡片采用 glassmorphism 效果（半透明背景，模糊）
- 添加 hover 时的 lift 动画（transform: translateY(-4px)）
- 使用 8px 基础间距系统
```

### 指定交互细节

详细描述用户操作和系统响应：

```
创建下拉菜单组件：
- 默认收起状态
- 点击触发按钮展开，200ms ease-out 动画
- 点击菜单外部区域自动关闭
- 选中项显示蓝色背景和白色对勾
- 支持键盘导航（上下箭头、回车选择、ESC 关闭）
```

## 实用模板

### 创建新页面

```
创建 [页面名称] 页面：

布局结构：
- [描述整体布局]
- [主要区域1]
- [主要区域2]

内容元素：
- [具体要显示什么]

样式要求：
- 配色：[颜色方案]
- 风格：[设计风格]

交互行为：
- [用户操作和响应]
```

**示例**：
```
创建产品展示页：

布局结构：
- 全宽的英雄区块在顶部
- 下方是产品网格，每行3个产品

内容元素：
- 每个产品显示图片、名称、价格、"查看详情"按钮

样式要求：
- 配色：主色为深蓝 #1e40af，强调色为橙色 #f97316
- 风格：现代简洁，大量留白

交互行为：
- 产品卡片 hover 时上浮并显示阴影
- 点击按钮跳转到详情页
```

### 添加功能

```
在 [位置] 添加 [功能名称]：

功能描述：
- [功能点1]
- [功能点2]

使用场景：
- [什么时候触发]
- [预期结果]

视觉呈现：
- [如何显示]
```

**示例**：
```
在文章列表页添加搜索功能：

功能描述：
- 顶部显示搜索框
- 输入关键词实时筛选文章
- 显示匹配结果数量

使用场景：
- 用户输入时即时搜索
- 搜索标题和内容
- 无匹配时显示"未找到相关文章"

视觉呈现：
- 搜索框宽度占页面70%，居中
- 带搜索图标和清除按钮
- 匹配的关键词高亮显示
```

### 修改样式

```
修改 [组件/元素] 的样式：
- 将 [属性] 从 [当前] 改为 [目标]
- 添加 [新样式]
- 调整 [布局相关]
```

**示例**：
```
修改导航栏样式：
- 将背景从白色改为透明渐变（顶部透明，向下渐变为白色）
- 添加模糊效果（backdrop-filter: blur(10px)）
- 调整高度从 60px 改为 80px
- Logo 尺寸增大到 40px
```

## 调试和修复

### 报告问题

清晰描述问题现象：

```
[功能] 出现问题：

预期效果：[应该怎样]
实际效果：[实际怎样]
重现步骤：[如何触发]

例如：
提交表单后未显示成功提示

预期效果：点击提交按钮后显示"提交成功"绿色提示
实际效果：没有任何提示，页面没有变化
重现步骤：填写表单 → 点击提交按钮
```

### 性能优化

```
优化 [页面/功能] 的性能：

当前问题：[具体问题]
优化目标：[期望改进]

例如：
优化图片加载性能：

当前问题：首页加载大量高清图片，首屏加载慢
优化目标：
- 实现图片懒加载
- 首屏图片优先加载
- 使用适当的图片尺寸
```

## 聊天模式 vs 实现模式

### 聊天模式（讨论方案）

当您想先讨论而不是立即实现时：

```
我想添加支付功能，我们先讨论一下最佳方案，
暂时不要修改代码
```

Lovable 会与您讨论不同选项，帮助您做决策。

### 实现模式（立即编码）

使用明确的动作词触发实现：

- "创建..."
- "添加..."
- "实现..."
- "修改..."
- "删除..."

## 常见场景示例

### 响应式设计

```
确保页面在所有设备上都有良好体验：
- 手机：单列布局，汉堡菜单
- 平板：两列布局
- 桌面：三列布局，完整导航栏
```

### 数据加载

```
实现文章列表的数据加载：
- 显示加载骨架屏（3个卡片）
- 从 API 获取数据
- 如果加载失败，显示错误提示和重试按钮
- 如果没有数据，显示"暂无文章"空状态
```

### 表单验证

```
为注册表单添加验证：
- 邮箱：必填，格式验证
- 密码：必填，至少8位，包含字母和数字
- 确认密码：必须与密码一致
- 实时显示错误提示（输入框下方红色文字）
```

## 最佳实践总结

1. **先整体后细节** - 先描述整体需求，再提具体要求
2. **一次一个功能** - 不要在一个提示中要求多个独立功能
3. **善用可视化编辑** - 简单样式调整用可视化编辑，节省积分
4. **清晰描述问题** - 遇到问题时详细说明现象和重现步骤
5. **测试后再继续** - 每个功能完成后测试，再添加新功能
6. **保存好的提示词** - 将有效的提示词模板保存下来重复使用

## 积分使用提示

- **聊天模式**：每条消息 1 积分，适合讨论和小改动
- **实现模式**：根据复杂度消耗积分，用于功能开发
- **可视化编辑**：直接编辑不消耗积分！

合理使用不同模式可以节省积分。
