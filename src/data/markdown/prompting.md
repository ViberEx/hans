# 提示词技巧

掌握提示词技巧是高效使用 Lovable 的关键。本指南将帮助您写出更好的提示词。

## 基本原则

### 1. 具体明确

❌ 不好的例子:
```
做一个表单
```

✅ 好的例子:
```
创建一个联系表单，包含姓名、邮箱、电话和留言文本框，
底部有提交按钮，使用卡片样式，添加表单验证
```

### 2. 分步进行

❌ 不好的例子:
```
创建一个完整的电商网站，包含首页、商品列表、详情页、
购物车、结账、用户中心、订单管理等所有功能
```

✅ 好的例子:
```
第一步：创建电商首页，包含导航栏和商品展示区
（后续迭代）第二步：添加商品列表和筛选功能
（后续迭代）第三步：实现购物车功能
```

### 3. 描述视觉效果

✅ 包含视觉细节:
```
创建一个英雄区块：
- 全屏高度
- 背景使用从蓝色到紫色的渐变
- 标题使用粗体，白色，字号 48px
- 添加一个白色边框的主按钮
- 整体居中对齐
```

## 高级技巧

### 提及参考设计

```
创建一个类似 Airbnb 风格的搜索栏：
简洁的圆角设计，悬浮阴影效果，
包含位置、日期和人数三个输入项
```

### 使用设计术语

```
创建一个仪表板布局：
- 使用 CSS Grid，3 列布局
- 卡片使用 Glassmorphism（毛玻璃）效果
- 添加 hover 时的 lift 动画（上浮）
- 间距使用 8px 的倍数
```

### 指定交互行为

```
添加一个下拉菜单：
- 默认收起状态
- 点击时展开，显示动画（200ms ease-out）
- 点击外部区域自动关闭
- 选中项显示对勾图标
```

## 常见场景的提示词模板

### 创建页面布局

```
创建 [页面类型] 页面：

布局：
- 顶部导航栏（固定定位）
- [描述主要内容区域]
- [描述侧边栏/其他区域]
- 底部页脚

样式：
- 使用 [颜色方案]
- 主色调 [具体颜色]
- [其他视觉要求]
```

### 添加组件

```
在 [位置] 添加 [组件类型]：

功能：
- [功能点 1]
- [功能点 2]

样式：
- [样式要求 1]
- [样式要求 2]

交互：
- [交互行为描述]
```

### 修改样式

```
修改 [元素/组件]：
- 将 [属性] 从 [当前值] 改为 [目标值]
- 添加 [新样式属性]
- 调整 [布局相关]
```

## 调试问题的提示词

### 报告错误

```
[功能/组件] 出现问题：
- 预期效果：[描述预期]
- 实际效果：[描述现状]
- 触发条件：[如何重现]
```

### 优化性能

```
优化 [页面/组件] 的性能：
- [具体性能问题]
- [期望改进的方面]
```

## 使用聊天模式

Lovable 支持 "聊天模式" - 只讨论不修改代码：

```
[以 "聊天模式" 开始]

我想了解如何实现 [功能]，
但先不要修改代码，我们先讨论一下方案
```

## 最佳实践总结

1. **先描述需求，再提细节**
2. **一次专注一个功能或区域**
3. **利用可视化编辑调整样式细节**
4. **遇到问题清晰描述现象**
5. **善用聊天模式进行规划**
